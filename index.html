The Deep Dive

Building Journey

From Copper Plates to Custom Neural Networks — the full story of the Smart Handwriting Learning Device.

What started as a competition entry for Innovate2Educate at WAVES 2025 evolved into a full-stack embedded AI product. Here's every architecture decision, failure, pivot, and breakthrough.

Phase 1

The Copper Plate Prototype

Hardware Brute-Force Approach

Our first prototype was purely electrical — copper plates with 3 contact pins per letter (Start, Mid, End). An Arduino Mega (54 digital I/O pins) read pin states while a child traces with a conductive stylus.

Each letter needed 3 GPIO pins, so 26 letters × 3 = 78 pins. An Arduino Uno drove an SPI-based TFT LCD for the UI. The two boards communicated via Serial.

// V1 Legacy Code: Scaling to Tamil (200+ chars)
// would need 600+ physical pins!

const int letterPins[numLetters][3] = {
  {22, 23, 24}, {23, 24, 25}, ...
};
❌ Fatal Flaw: Zero scalability — new languages = new copper plates

Phase 2

Web Canvas + Rule-Based Engine

Software Pivot with Geometric Features

Pivoted to a 4-wire resistive touchpad and built a browser-based canvas simulator (HTML5 Canvas + JS) to prototype recognition algorithms with 16 directional stroke features.

Features: 8 cardinal directions (U, D, L, R, UL, UR, DL, DR), geometric properties (isClosed, hasTopPeak, hasMiddleBar), start/end position in a 3×3 grid, and shape properties like aspectRatio.

Each letter had a hand-coded scoring function. E.g., Letter "A" gets +40 for hasTopPeak, +20 for diagonal strokes.

✅ ~80% accuracy for English A–Z
❌ Collapsed on Tamil — couldn't distinguish 'அ' vs 'இ'

Phase 3

1D Convolutional Neural Network

Deep Learning on Sequential Stroke Data

Designed a lightweight 1D CNN that processes raw (x,y) coordinate sequences — not images. Conv1D(32) → Pool → Conv1D(64) → Pool → Conv1D(128) → GAP → Dense(128) → Softmax.

Why 1D CNN, not 2D? A handwritten stroke is a time-series of (x,y) coordinates, not an image. 1D CNN processes the raw sequence → smaller model, faster inference, better generalization.

Preprocessing: Bounding-box normalization to [0,1] range, then arc-length resampling to exactly 64 evenly-spaced points using np.interp().

Data: ~7,000+ samples collected via web simulator, 3× augmentation, stratified 85/15 split.

✅ 98–99% validation accuracy on web canvas data

Phase 4

The Hardware Reality Check

ADC Noise, Pressure Variance & Domain Gap

Deploying to physical hardware revealed a critical domain gap — the same location produces different ADC values depending on stylus pressure. ESP32's 12-bit ADC was essential over Arduino's 10-bit.

The Problem: Raw coordinates clustered in a noisy band of ~2000–3999, not clean 0–1. Each coordinate averaged over 10 consecutive ADC samples (oversampling), with a 200 ADC threshold filtering false touches.

Why ESP32: The 4-wire resistive protocol requires rapidly switching GPIOs between OUTPUT and INPUT. Arduino Uno's GPIO 34/35 are INPUT-ONLY, breaking the protocol.

⚠️ Fix: Re-collected thousands of real touchpad samples & retrained

Phase 5

Wi-Fi Bridge & Flask Server

Networked Validation Pipeline

Built an intermediate networked version: ESP32 sends raw stroke JSON via Wi-Fi HTTP POST to a local Flask server running the TensorFlow model. Validated the AI pipeline end-to-end.

Architecture: ESP32 → Wi-Fi HTTP POST → Flask (TensorFlow .h5) → Response → Arduino Uno → TFT Display

This proved the concept worked but required a laptop running the server — not a standalone product.

⚠️ Works but requires laptop — need fully offline solution

Phase 6

True Edge AI — Final Architecture

Custom NumPy Inference Engine on Raspberry Pi

Eliminated TensorFlow entirely. Built a pure NumPy inference engine — reimplemented Conv1D, MaxPooling, Dense, ReLU, Softmax in raw linear algebra. Inference in <100ms on Pi 3B.

Weight Extraction: Custom script loads Keras .h5, extracts all weight matrices and bias vectors into a single .npz file.

NumPy Layers: Conv1D with manual zero-padding + sliding-window dot product. MaxPooling via strided maximum. GlobalAveragePooling = np.mean(x, axis=0).

Verification: Max output deviation vs TensorFlow: 0.000001 — mathematically identical.

Communication: USB Serial @ 9600 baud (115200 caused buffer overflow during audio playback). Audio via LM386 amplifier + Pi 3.5mm jack.

✅ Fully standalone, offline, boots as an appliance
